All signal providers on Syntiq Trading who provide advice on financial instruments (including but not limited to forex, synthetic indices, equities, commodities, or derivatives) must hold a valid FSP licence issued by the FSCA, or operate under the supervision of a licensed FSP (key individual). Syntiq Trading reserves the right to request proof of licensure at any time and to suspend any provider who cannot demonstrate compliance.

3
Proof of address
Bank statement or utility bill not older than 3 months, showing your name and residential address.
4
FSCA registration confirmation
Screenshot or PDF from the FSCA Financial Services Provider Register confirming your FSP number is active.

Signed declaration

Connect shared backend
Uses Google Sheets as database · Free · 5 minutes
Host the backend server on Render.com for free. Once running, every registration from any user appears in your admin panel in real time. Setup takes about 5 minutes.
1
Create a free GitHub account
Go to github.com and sign up for free if you don't have an account already.
2
Upload the server files to GitHub
1. On GitHub, click New repository → name it syntiq-backend → Public → Create
2. Click uploading an existing file
3. Upload both server.js and package.json from the server package you downloaded
4. Click Commit changes
3
Deploy on Render.com (free)
1. Go to render.com → sign up free with your GitHub account
2. Click New → Web Service
3. Connect your syntiq-backend GitHub repo
4. Set: Build command = npm install   Start command = node server.js
5. Click Create Web Service — wait ~2 minutes
6. Copy the URL shown (e.g. https://syntiq-backend.onrender.com)
4
Paste your Render URL below
The connection is tested live before saving. You will see a confirmation if it works.
